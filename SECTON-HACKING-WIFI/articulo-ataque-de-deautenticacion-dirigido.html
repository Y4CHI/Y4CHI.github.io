Ataque de Deautenticación dirigido

Un "ataque de desautenticación dirigido" en el contexto de la prueba de penetración (pentesting) de redes WiFi se refiere a un intento deliberado de desconectar a un usuario o dispositivo específico de una red inalámbrica al enviar paquetes de desautenticación falsos. Este tipo de ataque se utiliza para forzar a un dispositivo a desconectarse de una red WiFi legítima y, posiblemente, a intentar volver a conectarse.

Primeros identificamos que dispositivo vamos a desautenticas que este más activo, en este caso aremos uso del frame para poder ver cual es el más activo

Ejecutamos el siguiente comando para hacer el ataque de deatenticación dirigido

aireplay-ng -0 10 -e JEXXXXX -c EC:A1:D1:XX:XX:XX wlan0mon

-0 es el tipo de ataque que haremos

10 es el numero de paquetes que queremos enviar

-e es el nombre del ESSID

-c especificamos la mac del dispositivo a deatenticar

Si agregamos un 0 en el numero de paquetes que deseamos enviar nosotros estaremos enviando un numero ilimitado de paquetes que podremos parar con ctrl + c

Al finalizar la emición de paquetes podemos observar el HandShake
