Firewalls(Detección y Prevención)

Intrusion Detection Systems (IDS)

El sistema de detección de intrusos (IDS) es una capa (aplicacion) que inspeccionan la informacion de la aplicación (payload) de cada paquete que viaja por la red tratando de detectar cualquier ataque potencial, como conexiones remotas o impedir conexiones a ciertas paginas web.

Los IDS tan solo reportan actividades sospechosas, lo que hacen es enviar el reporte al firewall para que él se ocupe de bloquearlas.

Este tipo de firewalls comprueban las 7 capas del modelo OSI.

Un IDS, al igual que un antivirus, detecta el tráfico de riesgo mediante firmas. El proveedor proporciona actualizaciones frecuentes de firmas como tan pronto se encuentren nuevos vectores de ataque en internet.

El IDS no puede detectar algo si no lo sabe ya. También hay falsos positivos, suceden cuando hay tráfico legítimo marcado como malicioso.

Los IDS se dividen en dos categorias:

- Network Intrusion Detection Systems (NIDS)
- Host Intrusion Detection Systems (HIDS)

Intrusion Prevention Systems (IPS)

Los sistemas de prevención de intrusiones (IPS) pueden eliminar solicitudes maliciosas cuando la amenaza tiene una clasificación de riesgo superior a uno predefinido.

Durante las actividades de prueba de penetración, es posible que desee identificar si se utiliza un mecanismo similar a un cortafuegos en una red.
Si sospecha la presencia de un cortafuegos, es posible que desee comprobarlo para ver si existen anomalías en el TCP Three-Way Handshake que se introdujo previamente en este módulo.

Three-Way Handshake

Cuando hay un firewall, el siguiente comportamiento puede ser:
- Se envía TCP SYN, pero no hay respuestas TCP SYN/ACK
- Se envían paquetes TCP SYN pero se recibe una respuesta TCP RST/ACK

Firewall web: WAF

Es un tipo de firewall que supervisa, filtra o bloquea el trafico HTTP hacia o desde una app web.
Wafw00f es un programa para detectar WAF en servidores web.

        
          wafw00f http://apocalyst.htb
