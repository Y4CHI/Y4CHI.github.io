ARP Poisoning (Teoria Y Practica)

Teoria

ARP Poisoning es un poderoso ataque ya que puedes interceptar el trafico en una red

El atacante puede manipular las tablas ARP entre un host y el router dentro de una comunicacion, esto permitira hacer un ataque **Man in the middle (MITM)**.

En un ataque de envenenamiento del ARP, existen tres actores:

- Dos nodos en la red (clientes, servidores, routers, impresoras,…)

- El atacante

El atacante puede manipular la cache dentro de las tablas del ARP de cada host para enviar respuestas “gratuitas” de ARP.

Estas respuestas “gratuitas” son mensajes ARP sin solicitud, en otras palabras, el atacante envia respuestas sin esperar a que el host realice una solicitud

El atacante explota mensajes gratuitos de ARP modificando la MAC de un host para engañarlo y hacerse pasar por él.

Este engaño se le debe de aplicar a las dos victimas.

Tan pronto como la tabla de cache ARP contenga la informacion falsa, cada paquete de cada comunicacion entre los dos nodos envenenados se enviaran a la maquina del atacante.

El atacante puede evitar que la entrada envenenada caduque enviando respuestas ARP gratuitas cada 30 segundos más o menos.

En el momento que el atacante recibe los paquetes, esté los reenvia al destino correcto para no levantar sospechas.

Esta actividad puede llegar mas lejos ya que el atacante puede cambiar el contenido de los paquetes.

Practica

Dsniff es una coleccion de herramientas para auditoria y pentesting, incluye arpspoof, una utilidad hecha para interceptar el trafico en una red LAN.

Antes de empezar con la herramienta, debemos de activar el “Linux Kernel IP Forwarding", una caracteristica que **transforma un Linux en un router**.

Para activarlo, le decimos a nuestra maquina que reenvie los paquetes que interceptemos al destinatario real.

  
    echo 1 > /proc/sys/net/ipv4/ip_forward

Podemos hacer funcionar arpspoof con:

  
    arpspoof -i <interface> -t <target> -r <host>

Si queremos interceptar el trafico entre 192.168.4.11 (cliente) y 192.168.4.16 (servidor) debemos de ejecutar los siguientes comandos:

    
      echo 1 > /proc/sys/net/ipv4/ip_forward
      arpspoof -i eth0 -t 152.168.4.11 -r 192.168.4.16

Ahora podemos ejecutar Wireshark e interceptar el trafico.

Si filtramos por HTTP podemos ver las solicitudes POST (inicios de sesion) que se han hecho en las paginas web de la victima.

Tambien podemos filtrar por SMB y ver los archivos que ha abierto el usuario en una red compartida.
En la barra de menus entramos en File > Export Objetcts > SMB/SMB2

Nos aparecera una ventana con la lista de objetos (archivos) que se han enviado por la red.

Nos centramos en los que su contenido es de tipo “FILE”, lo seleccionamos y le damos al boton “Save as” para guardarlo en nuestro equipo.

Podemos identificarlo con el comando “file” de linux, en este caso es una imagen que la victima ha abierto dentro de su recurso de red.
