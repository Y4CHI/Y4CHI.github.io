Como instalar Node en la terminal de Linux usando nvm

Node.js es un entorno en tiempo de ejecucion multiplataforma para la capa de servidor, orientado en un entorno de ejecucion de javascript con eventos asìncronos, esta diseñado generalmente para crear aplicaciones escalables , con la cual tambien nos permite gestionar multiples conexiones al mismo tiempo, actualmente muchas marcar reconocidas usan Node.js como Microsoft, Linkedin, Netflix, Ebay, Uber, etc.

¿Que es nvm?

nvm(Node Version Manager) es un gestor de versiones de Node.Js ,la cual nos permite realizar operacion como desistalar , instalar , cambiar versions de Node.Js.

Instalacion de Node.js

Primero antes nada tendremos que descargarnos el gestor de versiones de Nodejs para poder escoger nuestra version a instalar e utilizar. 

         
  curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.35.3/install.sh | bash

Despues de habernos descargado en nvm nos la instalamos con los siguientes comandos, con el parametro --lts estamos escogiendo un soporte de largo plazo el Long Term support , la cual es una version que actualmente se encuentra con soporte por un periodo medianamente de mas de 10 meses, tambien podriamos instalarnos una version diferente, con simplemnete reemplazando el "--lts" con la version de NodeJs que deseas instalar. 

         
  nvm install --lts
            nvm install 16 #Se instalara la ultima version 16 de Node.js
          nvm install 16.20.2 #Se instalara la version 16.20.2 de Node.Js

Ya que ahora ya lo tenemos instalado la version en este caso instalamos la version 16.20.2 de NodeJs, asi que solo nos falta configurarlo para usar la version que hemos instalado con un sencillo comando, tambien podremos listar nuestras versiones instaladas. 

           
    nvm use 16.20.2 # Usamos la version 16.20.2 de Node.js
    nvm ls #Lista las version de Node instaladas.

Creamos un proyecto en React

Despues de hacer todo lo anterior , ya tendremos listo para poder crear nuestro proyecto web, en este caso para probar si todo funciono correctamente nos creamos una aplicacion en react, usando npx que nos facilitara crear nuestro proyecto. 

           
  npx create-react-app MiPrimerProyecto

porque no usar npm en vez de npx, la diferencia no es mucha en realidad la 'x' significa execution , asi que lo que hace despues de terminar de crear nuestro proyectos nos ejecuta nuestro proyecto y eso nos facilita la configuracion , asi que ahora solo nos falta ejecutar nuestra aplicacion web.

             
  cd MiPrimerProyecto #Ingresamos a la carpeta donde se encuentra nuestros archivos
  npm start #Ejecutamos nuestra aplicacion y nos crea un servidor en puerto 3000
