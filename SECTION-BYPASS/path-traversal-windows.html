Path Traversal en Windows

En el vasto mundo de la ciberseguridad, es esencial comprender las amenazas que pueden comprometer la integridad de nuestros sistemas. Uno de estos peligros, conocido como "Path Traversal," es una técnica sigilosa que puede abrir las puertas de nuestro sistema Windows a los atacantes. En este artículo, exploraremos en qué consiste el Path Traversal, cómo funciona en el entorno de Windows. Acompáñanos en este viaje por las rutas peligrosas del ciberespacio.

En sistemas operativos Linux sabemos que las rutas son conformadas por / (slash o barra), en este articulo usaremos el termino barra para referirnios a / , pero en sistemas Windows las rutas estan conformadas por barras invertidas \ .

Linux:

/home/usuario

Windows:

C:\Users\black

Durante un ataque de Path Traversal en linux usamos la siguiente ruta para validar si es vulnerable:

/etc/passwd

Pero en windows los archivos a los cuales podemos apuntar son:

C:\Windows\System32\drivers\etc\network.txt

C:\ProgramData\ssh\sshd_config

C:\Windows\System32\drivers\etc\hosts

C:\xampp\apache\conf\httpd.conf

Este vector no está disponible en el entorno de Windows y lamentablemente, no existe una solución directa para esta situación. Además, identificar archivos que contienen información sensible en Windows puede ser un desafío, ya que no es sencillo acceder al contenido de los directorios. Por lo tanto, para descubrir archivos que contengan datos confidenciales, debemos llevar a cabo un análisis exhaustivo de la aplicación web y recopilar información sobre el servidor web, el marco de trabajo utilizado y el lenguaje de programación empleado.

Bien una vez identificada una aplicativo web usado podemos buscar su documenetación con la finalidad de encontrar las rutas de archivos de sensibles, configuraciones, etc.

Como anterior mente vimos para sistemas linux usamos ../ y en sistemas windows ..\ , pero la especificación que establece que las URLs deben usar barras ("/") en lugar de contrabarras ("") en su estructura es la RFC 3986, titulada "Uniform Resource Identifier (URI): Generic Syntax". Esta RFC describe la sintaxis general de los Identificadores Uniformes de Recursos (URI), que incluyen las URL (Uniform Resource Locators). En la RFC 3986 se detallan las reglas y convenciones para la composición de las URL, y establece que las barras ("/") deben utilizarse como delimitadores en la estructura de las URL . En otras palabras tambien es conveniente probar con ../ en sistemas operavitos Windows.

BYPASS

En muchas ocaciones el corta fuegos detecta ../ como unos caracteres maliciosos para evadir esta detección podemos usar la codificación de caracteres, codificaremos los puntos en formato ASCI %2e y al final la ruta quedará así:

%2e%2e/%2e%2e/%2e%2e/%2e%2e/%2e%2e/%2e%2e/%2e%2e/etc/passwd

Y huala logramos evadirlo!
